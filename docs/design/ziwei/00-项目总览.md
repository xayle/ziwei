# 紫微斗数模块 — c2 项目总览

> 本文档描述 **c2 / 浮生** 仓库内的紫微引擎与前端集成，非第三方 iztro 项目说明。

## 1. 定位

c2 紫微模块与八字引擎并列，服务于：

- 标准十二宫安星与运限（大限 / 流年 / 流月）
- 与档案（出生时间、经度、性别）一致的 API 排盘
- Fusheng 新前端命盘展示与 Report 双模块输出

成功标准：**盘面坐标可回归、缺失字段显式标注、与八字档案共用真太阳时开关**。

## 2. 架构分层

```text
frontend/src/
├── api/ziwei.ts              # 请求/响应类型与 API 客户端
├── components/fusheng/       # FushengZiweiPlate、Report 图表
├── components/ziwei/         # 完整工作台组件（时间轴、设置等）
├── stores/fushengReport.ts   # 与八字共用的报告缓存
└── utils/buildChartRequests.ts  # 档案 → 紫微/八字请求

services/ziwei_engine/        # Python 安星与运限核心
├── __init__.py               # ziwei_full 主入口
├── patterns.py               # 格局规则
├── forecast.py               # 叠宫运势
└── structural_summary.py     # 命身宫、借星结构摘要

routers/ziwei.py              # FastAPI 路由与 _chart_to_response
data/ziwei_ground_truth.json  # ZW01–ZW08 黄金盘回归
```

## 3. 数据流

1. 用户在 **Profile** 填写出生时间、经度、真太阳时开关。
2. `buildZiweiRequest()` 将档案转为 `ZiweiRequest`（`solarTime=true` 时传入 `longitude`）。
3. `POST /api/v1/ziwei/compute` 调用 `ziwei_full()`，经 `_chart_to_response` 返回 `PalaceResponse[]`。
4. Fusheng 页面消费 `FushengZiweiPlate`；完整工作台使用 `components/ziwei/*`。

## 4. 关键字段

| 字段 | 说明 |
|------|------|
| `life_palace_gz` / `body_palace_gz` | 命宫、身宫干支 |
| `palaces[].is_body_palace` | 身宫所在宫位（引擎内标注） |
| `palaces[].borrowed_*` | 空宫借对宫主星 |
| `true_solar_time` | 经度修正后的时刻（未传经度则为空） |
| `patterns` | 格局列表（含 rule_id / source） |
| `missing_fields` | 引擎未能计算的字段 |

## 5. 回归与黄金盘

- **ZW01–ZW08**：`data/ziwei_ground_truth.json`，主星/辅煞坐标锁定
- **双验档案**：`data/dual_verify_cases.json`（八字四柱 + 紫微命宫对照）
- **测试**：`tests/test_ziwei_engine.py`、`tests/test_ziwei_golden_regression.py`

## 6. 与八字的关系

- 共用 Profile 档案与 `fushengReport` store
- 真太阳时：`buildBaziRequest.solar_time_enabled` 与 `buildZiweiRequest.longitude`（条件传入）一致
- Report 分章展示，避免混排
