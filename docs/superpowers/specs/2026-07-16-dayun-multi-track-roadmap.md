## 0. 一句话

把「大运叙事」从墙文拆成**可规划的命理读面**：上半是**可核对数据**，下半是**分域讲解**；在八字启发式讲清楚的前提下，用**河洛、奇门**作对照分栏（互证），而不是第二本胡说八道长文叠上去。

## 1. 背景与约束

### 1.1 现状

| 能力 | 状态 |
|------|------|
| 八字大运干支 / 十神 / 顺逆 | 引擎有 |
| 大运叙事模板（事业/财运/情感/健康/古籍） | 有，但拼成单字符串墙文 |
| 河洛易数引擎 | **无**（语料偶见，如滴天髓注疏） |
| 奇门遁甲引擎 | **无**；资料目录标「产品路线外」；POST-W14 / PLATFORM 曾写明不做 |
| 盲派体系 | **无独立引擎**；勿与八字启发式混称「盲派推算」 |

### 1.2 产品决议（本次）

- **主轨**：子平八字大运（数据条 + 人生域讲解）。
- **参考轨**：河洛、奇门 → 并列对照，措辞为「互证 / 另法视角」，**不得改写**主轨结论。
- **不做**：术数超市入口堆砌；大师撮合；用 LLM 冒充奇门排盘。

### 1.3 与旧路线冲突

`FUSHENG-EXECUTION-PRIORITY-POST-W14.md` 将奇门列为「明确不做」。本规划若通过，应新增变更记录：**允许「参考轨」分期研发，仍不做奇门全科产品化主流程**。

## 2. 信息架构（目标读面）

每个大运步（例：丙子 · 0–9 岁）一张「运步卡」：

```
┌──────────────────────────────────────────────────────────┐
│ A. 数据条（只放可核对事实）                                  │
│    干支 · 岁运 · 十神 · 用神顺逆 · 格局/用神引擎摘要            │
│    来源徽章：引擎                                           │
├──────────────────────────────────────────────────────────┤
│ B. 讲解 · 八字主轨（结构化，非墙文）                           │
│    总起（core）                                             │
│    ┌──────┐ ┌──────┐                                       │
│    │ 事业 │ │ 财运 │                                       │
│    ├──────┤ ├──────┤                                       │
│    │ 情感 │ │ 健康 │                                       │
│    └──────┘ └──────┘                                       │
│    脚注：子平古籍佐证 + 免责声明 · 徽章：启发式                │
├──────────────────────────────────────────────────────────┤
│ C. 互证参考（默认折叠；点开才读）                             │
│    Tab / 分段：河洛易数 | 奇门遁甲                            │
│    每轨内部再分：【排盘/数理数据】|【解读】|【出处】            │
│    未接入时：明确「本轨未计算」+ 计划版本，禁止灌水             │
└──────────────────────────────────────────────────────────┘
```

**硬规则**

1. A 与 B 永不混排成一段散文。  
2. C 不得用与 B 同款「事业 400 字」模板灌屏；参考轨以**短对照 + 出处**为主。  
3. 任一轨缺引擎 → UI 走 `ResultState` / Trust「未计算」，不沉默降级装有。

## 4. 分期路线图

### P0 — 八字结构化读面（方案 B，可独立上线）

**目标：** 消灭墙文；数据 / 讲解分流。

| 项 | 内容 |
|----|------|
| BE | `narrative_sections`（core/career/wealth/love/health/classics/disclaimer）；保留 `narrative` 兼容 |
| FE | 运步卡 A+B；启发式徽章；按需加载不变 |
| 验收 | 四域可见；无整页散文主容器；旧客户端仍可读 `narrative` |
| 河洛/奇门 | UI 预留 C 区折叠壳，文案：「互证参考 · 即将接入」，**不出假数** |

详见：[dayun-narrative-structured-design](./2026-07-16-dayun-narrative-structured-design.md)（本规划吸收其非目标变更：参考轨改为分期做，不再「永久不做」）。

### P1 — 主轨可信度打磨

| 项 | 内容 |
|----|------|
| 古籍 | classics 带来源页码 / `verification_status` 与全站典籍轨对齐 |
| Trust | 大运卡写入 provenance 短摘要（模式、子时日界等人读） |
| 报告 | `ReportView` 卷三同源 sections，避免只改八字页 |
| 文案 | 总起与四域字数上限写死（每域短读，忌再拼回 600 字墙） |

### P2 — 河洛互证参考轨（最小可用）

| 项 | 内容 |
|----|------|
| 范围 | **参考轨**，不是河洛全科产品 |
| 最小引擎 / 规则包 | 与出生年命或大运干支可对齐的**可复述规则**（例：数理落宫/五行生克摘要——细则另开 `heluo` ADR）；输出固定结构见 §5 |
| 无完整引擎前 | 仅允许：结构化「规则说明 + 典籍摘句 + 未计算字段」；**禁止** LLM 代填河洛数 |
| UI | C 区 · 河洛 Tab：数据小表 + ≤3 条对照句 + 出处 |
| 验收 | 与八字结论冲突时并列展示，不抹平；有 `layer` / missing |

### P3 — 奇门互证参考轨（最小可用）

| 项 | 内容 |
|----|------|
| 范围 | 大运/流年节点的**态势互证**，非奇门完整排盘工作台 |
| 前置 | 选定时空盘口径（拆补 vs 转盘等）写进 ADR；出生地/真太阳时复用档案 |
| 输出 | 值符/值使/格局类**短标签** + 一句互证 + 出处；长篇奇门课外包死 |
| 注册 | 若走引擎注册表，单独 method_registry；Trust 必标「奇门参考」 |
| 验收 | 与 POST-W14「不做奇门全科」不冲突：无独立超市入口亦可 |

### P4 — 可选增强（本规划不排期绑死）

- 运步卡内「八字 vs 河洛 vs 奇门」一张对照表（仅标签级）  
- PDF 导出分栏  
- 用户开关：默认折叠互证轨

## 5. 数据契约（演进）

### 5.1 P0（八字）

```text
DayunStep {
  …engine fields…
  narrative?: string
  narrative_sections?: {
    core, career, wealth, love, health,
    trend_note?, classics[{source,text}], disclaimer
  }
}
```

### 5.2 P2+（互证容器）

```text
DayunStep {
  …
  reference_tracks?: {
    heluo?: TrackPayload | null
    qimen?: TrackPayload | null
  }
}

TrackPayload {
  status: "ok" | "not_implemented" | "degraded" | "missing_input"
  layer: "engine" | "classical" | "heuristic" | "modern_convention"
  data: Record<string, string | number>   // 短键值，人读在 FE 映射
  notes: string[]                         // ≤3 条对照句
  classics: Array<{ source, text, verification_status? }>
  disclaimer: string
}
```

`status=not_implemented` 时 FE 只渲染占位，不解析假 `data`。

## 6. Trust / 合规

| 层 | 规则 |
|----|------|
| 主轨讲解 | 固定「启发式」；声明不构成预测建议（沿用现 disclaimer） |
| 古籍 | 与全站典籍核验状态一致，禁伪造页码 |
| 参考轨 | 标题必须含「互证参考」；冲突时双轨并列 |
| 缺失 | 走既有 missing / ResultState，不落到主轨散文里 |

## 8. 风险

| 风险 | 缓解 |
|------|------|
| 参考轨做成第二堵墙文 | 硬限制 notes≤3、禁套用八字四域长模板 |
| 未开工却展示「已排奇门」 | `not_implemented` 为唯一合法空态 |
| 范围膨胀成术数超市 | 本规划无独立奇门/河洛首页；只挂大运卡 C 区 |
| 与现 CI「产品不做奇门」文档冲突 | DOC-01 与发布说明同步 |

---

## 9. 成功标准（产品）

1. 用户能在 10 秒内从数据条读出「哪步运、十神、顺逆」。  
2. 讲解四域可扫读，不再出现单段墙文作为默认态。  
3. 打开互证后，能说出「这是另一方法的对照」，且知是否已计算。  
4. 河洛/奇门未上线时，产品不假装有数。
