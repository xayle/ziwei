## 1. 要解决的现状（问题清单 → 验收标准）

| # | 现状 | 目标态（验收） |
|---|------|----------------|
| S1 | 引擎事实与 Trust 藏在折叠/深读 | **速览首屏**可见：盘面主角 + 一句「可核对洞察」+ 缺失/风险一条；Trust 有摘要入口，不全进深读才看见 |
| S2 | 主读路径偏模板长文 | 默认路径**不以**启发式墙文开场；长文仅在深读/按需加载后出现，且分块 |
| S3 | 首屏完成度自评约 4/10 | 八字/报告试读路径达到「10 秒能说出这盘要点」；内部目标 **≥7/10**（对照 MARKET-ENTRY） |
| S4 | 大运叙事墙文消解差异点 | 大运：数据条 + 四域讲解 + 古籍脚注；无单段落主容器（方案 B / 多轨 P0） |
| S5 | 叙事反了（结构藏、文案抢镜） | 信息优先级固定：**事实 > 结构讲解 > 启发式 > AI 问书** |
| S6 | 河洛/奇门无引擎却想上 | **本期不做算法**；仅允许「互证未接入」占位，禁止灌水 |

**明确不在本方案一次做完：** GTM 全开（R109 A）、奇门/河洛真引擎、术数超市、达人撮合。

---

## 2. 产品原则（写进评审口径）

1. **结构先于文案**：用户先看见干支/格局/用神/缺失，再看见「怎么办」类句子。  
2. **数据 ≠ 讲解**：同一组件内上下分区，禁止拼成一段散文。  
3. **敢标缺失是卖点**：missing / 双轨差 / 降级横幅进主路径摘要，不进「小字附录」才算完成。  
4. **启发式降权**：徽章必显「启发式」；默认折叠或按需；禁止与典籍同视觉权重。  
5. **无引擎不装有**：河洛/奇门 `not_implemented`，零假数。

## 4. 刀 1 — 吸引补丁：结构上前台

### 4.1 范围页

| 页 | 改动重点 |
|----|----------|
| 八字 `NewBaziView` 速览 | Cover 下增加 **Structure Insight Strip**（见下） |
| 报告 `ReportView` 卷一入口 | 同款：一句引擎事实 + 互证/缺失 tip |
| （可选）首页续读卡 | 若有档案，展示「上次盘：日主·格局」而非空诗句 |

### 4.2 Structure Insight Strip（必须有）

从已有 API/前端计算拼出，**禁止新 LLM**：

| 槽位 | 内容示例 | 来源 |
|------|----------|------|
| 主角 | 庚辰 · 正印格 · 用神火土 · 中和 | pillars / geju / yongshen / strength |
| 一句洞察 | 「月令印星透干，喜火土调候」类 **短句**（≤40 字） | 优先 `evidence_chain` / geju note / 规则命中首条人读；勿贴整段 narrative |
| 可信提示 | 缺字段一条 **或** 双轨/校验 tip | missing_fields / validation / provenance 人读 |
| CTA | 「看结构校勘」→ `structure`；「按需加载大运」→ deep+dayun | 深度切换已有 |

### 4.3 速览禁止项

- 不在速览展开完整 `AnalysisPanel` 启发式长文。  
- 不在速览默认展开大运叙事。  
- Trust：保留「校勘 N 项」摘要；点进结构档看全表（可强化现有 overview 摘要，不必新发明 L1–L4 文案）。

### 4.4 分享切片（为 GTM 预埋，本刀可做最小版）

- 竖版/卡片预览优先：**盘面区截图逻辑 + 一句洞察**（沿用已有 Douyin/分享能力则接线；无则只保证 DOM `data-testid` 可截）。  
- **禁止**默认分享大运墙文。

### 4.5 验收

- [ ] 空档案/就绪档案：八字速览首屏 10 秒内可读出日主·格局·用神  
- [ ] 有 missing 时首屏可见人读提示（非英文键）  
- [ ] 速览 DOM 无「规则与提示」整块长文主露（深读才有）  
- [ ] 单测 / 一条 E2E：`bazi-structure-insight` 可见

## 5. 刀 2 — 大运 P0：墙文拆分（方案 B）

完整契约见 [dayun-narrative-structured-design](./2026-07-16-dayun-narrative-structured-design.md)。摘要：

| 层 | 内容 |
|----|------|
| API | `narrative_sections` + 兼容 `narrative` |
| UI | 数据条 / 四域讲解 / 古籍脚注；徽章启发式 |
| 互证壳 | 折叠「河洛｜奇门 · 未接入」占位（多轨规划 C 区） |
| 报告 | 卷三同源改，避免只修八字页 |

### 验收

- [ ] 加载大运后无单段落墙文作为主读  
- [ ] 事业/财运/情感/健康分块可见  
- [ ] 互证区存在且为 `未接入`，无假排盘

## 6. 刀 3 — 路径收敛：默认读法

| 项 | 决议 |
|----|------|
| 八字默认深度 | 保持 **速览**；但速览必须带刀 1 的 Strip（现在速览过空/过弱） |
| 进入深读 | 需显式点击；深读内大运仍 **按钮按需加载**（不自动 LLM/不自动 narrative） |
| 报告默认 | 连续读时卷一结构优先；卷五推断默认折叠（现状保持并审计） |
| 卷六 AI | 保持折叠/手动；Mock 必须可区分（若尚未，本刀审计补洞） |

### 验收

- [ ] 新用户路径：建档 → 八字速览洞察 → 结构 →（可选）大运，无「一进来墙文」  
- [ ] 试读 E2E（既有 trial-read）不回退；可追加 insight 断言

## 9. 成功指标（产品）

| 指标 | 基线（质感） | 目标 |

| 首屏「能说出要点」 | 偏弱 / 4 分感 | 内部盲测 ≥7 分感 |
