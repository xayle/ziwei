# 代码现状审计与整理计划

## 目标
以代码、路由、页面、测试为事实来源，完成当前项目功能盘点、质量基线校准、技术债排序，并形成分批整理方案。

## Phase 1：建立事实基线
1. 以后端路由总入口为模块地图，梳理已挂载业务域。
2. 以前端路由为页面地图，梳理 SPA 已交付页面。
3. 对照 legacy static 页面，标记仍在使用、迁移中、可下线内容。
4. 输出系统真实模块清单。

## Phase 2：端到端功能盘点
1. 按业务域建立功能矩阵。
2. 每项功能标注：
   - 前端入口
   - API
   - 服务/引擎
   - 测试/文档
3. 重点覆盖：
   - 八字
   - 紫微
   - 姓名
   - 西占
   - 塔罗
   - 择日
   - 风水
   - 报告书
4. 标记状态：
   - 已完整
   - 可用但不稳定
   - 仅前端/仅后端
   - legacy 残留

## Phase 3：校准质量与运行基线
1. 重新核对真实测试、类型检查、构建、关键页面运行状态。
2. 区分“当前真实问题”和“历史已关闭问题”。
3. 输出真实质量基线表。

## Phase 4：梳理架构与技术债
1. 后端：
   - 检查入口职责过载问题
   - 检查数据库初始化与迁移双轨问题
2. 前端：
   - 检查超大页面文件
   - 检查路由保护和权限边界
3. 输出结构性技术债清单并排序。

## Phase 5：统一文档口径
1. 区分：
   - 现状文档
   - 历史复盘
   - 路线图
   - 已完成记录
2. 明确以后以代码和测试为准。
3. 输出文档可信度表。

## Phase 6：形成整理优先级
### 第一优先级
- 模块地图
- 功能矩阵
- 质量基线
- legacy 边界清理
- 文档口径统一

### 第二优先级
- 拆分后端超级入口
- 拆分前端超大页面
- 收敛模块边界

### 第三优先级
- lint 分层治理
- 数据库迁移治理
- 权限一致性治理

## Phase 7：执行批次
### 批次 A
- 现状文档
- 功能矩阵
- 质量基线
- legacy 清单

### 批次 B
- 入口拆分
- 超大文件拆分
- 模块边界收敛

### 批次 C
- 测试门禁
- lint 治理
- 迁移治理
- 权限一致性

## 核心原则
1. 以代码、路由、页面、测试为事实来源。
2. 先解决认知混乱和边界不清，再做大改。
3. 先补护栏，再改结构，再回归验证。
4. 优先围绕八字、紫微、报告书、工作台、审核、LLM 主链路推进。
