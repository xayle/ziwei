# v5.3.1（2026-02-25）

## 新增
- 请求关联（request_id）：当请求头包含 X-Request-Id 且有效时优先沿用；否则服务端生成 UUIDv4。会同时在响应体字段 request_id 与响应头 X-Request-Id 中返回。
- 保护策略（Guardrails）：
  - X-Request-Id 含非法字符：替换为 UUID，并在 validation.warnings 中发出 request_id_invalid_chars
  - X-Request-Id 长度超过 128：截断，并在 validation.warnings 中发出 request_id_truncated
- 非阻塞 validation.warnings：告警字符串统一为 code: key=value ... 格式（例如：tz_mismatch: dt_offset=+09:00 tz=Asia/Shanghai action=tz_ignored_for_aware_dt），用于提示 aware dt 与传入 tz 不一致的情况（不影响计算）。

## 兼容性
- 不改变任何计算逻辑；现有客户端保持兼容（新增字段/告警为可选消费）。

## 网关 / 前端注意事项
- 确保网关/反代不会剥离 X-Request-Id 响应头。
- 若浏览器前端需要读取该响应头，请在 CORS 中暴露：Access-Control-Expose-Headers: X-Request-Id。
