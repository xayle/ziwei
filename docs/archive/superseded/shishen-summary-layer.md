# 八字十神汇总层

## 目标

把八字的十神信息从“只返回四柱映射”升级成“可直接展示、可直接解释”的结构化汇总层。

## 输出结构

新增 `shishen_summary`，字段含义如下：

- `day_stem`：日主天干
- `day_element`：日主五行
- `day_yinyang`：日主阴阳
- `pillars`：四柱十神映射，包含年/月/日/时四个柱位
- `score_breakdown`：十神总分
- `score_share`：十神占比
- `dominant`：主导十神
- `hidden_contrib_by_ten_god`：藏干十神贡献汇总
- `contributions`：逐条贡献明细，区分天干与藏干
- `liuqin_summary`：六亲快速摘要
- `summary_text`：给前端直接展示的概括文本

## 设计原则

- 不改底层十神判定逻辑，只做结构化包装。
- 日柱保留为 `日主`，不强行改成十神名。
- 藏干贡献单独列出，方便解释“为什么这个十神占比高”。
- 对前端来说，这一层是展示入口，不再需要自己拼分布。

## 接口影响

- `POST /api/v1/bazi/full`
- `POST /api/v1/verify`

这两个接口都会继续保留旧的 `ten_gods`，同时新增 `shishen_summary`。

## 展示建议

- 第一屏优先显示：日主、主导十神、十神占比前 3。
- 再显示：四柱十神、藏干贡献、六亲摘要。
- 对缺失字段直接标记，不隐藏。

## 备注

这不是新算法，只是把现有十神、藏干、五行、六亲信息收拢成统一的响应层。
