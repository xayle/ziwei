# 个人档案型命理产品前端目标与布局基线 v1.1

日期：2026-06-17
状态：前端规划基线

## 1. 总目标

把当前项目收敛成一个稳定的 `个人档案型命理产品`。

核心链路固定为：

`基础信息输入 -> 生成个人数据卡 -> 输出八字 / 紫微 / 姓名结果 -> 形成正式 PDF 档案 -> 支持人工注释讲解`

## 2. 产品定位

这个产品应当被理解为 `档案交付工具`，不是：

- 聊天机器人
- 玄学社区
- 娱乐测试页
- 门户式命理导航站
- 多入口入口堆叠

用户最终拿到的内容，应该像一份可交付、可讲解、可留档的正式文件。

## 3. 前端设计原则

1. 首页只保留输入和预览。
2. 档案页作为标准化资料编辑中心。
3. 报告页作为正式交付件。
4. 所有页面统一为一套正式、克制、档案感视觉语言。
5. 所有输出遵循 `先摘要、再结构、后解释`。
6. 长文本拆短，复杂内容分层，不在一个区块里塞完。
7. 保留底层命理计算逻辑，不做大一统重写。
8. 报告页预览与 PDF 章节结构保持同构。

## 4. 页面职责边界

### 4.1 首页

首页只做三件事：

1. 填基础信息。
2. 预览个人数据卡。
3. 进入报告主链路。

首页不承载：

- 八字完整分析
- 紫微完整分析
- 姓名完整分析
- 复杂入口堆叠
- 次级工具的并列展示

### 4.2 档案页

档案页是基础资料编辑中心，职责是把输入标准化。

它要解决的问题是：

- 基础信息是否完整
- 时间信息是否可信
- 地点信息是否标准化
- 现居地和历史事件是否可复用

档案页不承担正式报告内容。

### 4.3 报告页

报告页是正式交付件，职责是把计算结果整理成可阅读的档案。

它要呈现：

- 封面
- 目录
- 基础档案
- 时间轴
- 八字总览
- 紫微总览
- 姓名分析
- 综合总结
- 人工注释区
- 附录与免责声明

报告页不能再像控制台。

## 5. 首页布局

### 总体结构

- 顶部 Hero
- 左侧输入区
- 右侧个人数据卡预览
- 底部主操作区

### 左侧输入区

按四组组织：

1. 基础信息
2. 时间设置
3. 现居地
4. 历史事件 / 当前关注

### 右侧预览区

按三层展示：

1. 基础档案摘要
2. 时间与地点摘要
3. 现实标签与状态摘要

### 底部按钮

只保留一个主按钮：

- `生成档案并进入报告`

次级按钮尽量少，避免首页入口混乱。

## 6. 档案页布局

### 总体结构

- 顶部说明
- 中间编辑区
- 右侧摘要卡
- 底部保存 / 重置 / 前往报告

### 编辑区

分成五块：

1. 基础信息
2. 时间与地点
3. 现居地
4. 分析偏好
5. 历史关键事件

### 右侧摘要

固定显示：

- 档案完整度
- 时间可信度
- 当前关注重点
- 关键标签
- 当前补全状态

## 7. 报告页布局

### 总体结构

保持正式交付感的三栏结构：

- 左侧导航 / 章节索引
- 中间正文
- 右侧摘要 / 注释 / 预警

### 章节顺序

1. 封面
2. 目录
3. 个人基础档案
4. 时间轴
5. 八字总览
6. 紫微总览
7. 姓名分析
8. 综合总结
9. 人工注释区
10. 附录与免责声明

### 章节原则

- 每章先给摘要
- 再给结构信息
- 最后给解释
- 不把原始计算过程直接铺满正文

## 8. 视觉系统

### 颜色

- 暖米白
- 浅砂金
- 深墨色
- 低饱和青绿色

### 语义

- 金色：主状态
- 青绿色：正常状态
- 红色：仅用于风险或预警

### 风格关键词

- 档案
- 报告
- 纸张
- 克制
- 可信

### 禁止方向

- 紫色霓虹
- 玄学闪光
- 娱乐插画
- 门户站密集排版
- 工具台式信息堆叠

## 9. 内容语法

统一所有页面和章节的表达方式：

1. 先结论
2. 再结构
3. 后解释

对应到页面里：

- 首页的预览卡只放摘要
- 档案页的右侧卡只放状态和关键信息
- 报告页的正文才进入完整分析

## 10. 数据分层

### 基础信息层

包括：

- 姓名
- 性别
- 出生年月日
- 出生时间
- 出生地
- 时区
- 经度
- 历法
- 闰月
- 时辰精度
- 未知时辰兜底
- 真太阳时
- 现居地
- 当前关注重点
- 历史事件

### 计算结果层

包括：

- 八字摘要
- 紫微摘要
- 姓名摘要

### 解释层

包括：

- 时间可信度
- 关键年份
- 风险提示
- 人工注释
- 综合结论

## 11. 当前实施约束

1. 保留 Vue + Vite + Router 技术栈。
2. 不迁移框架。
3. 不重写底层命理计算逻辑。
4. 优先改页面结构和视觉语法。
5. 报告页和 PDF 预览尽量保持同构。

## 12. 当前推进顺序

### 第 1 步：首页重做

- 只保留基础输入和个人数据卡预览。
- 主按钮收口到 `生成档案并进入报告`。
- 不在首页塞完整分析结果。

### 第 2 步：档案页重做

- 把档案页做成唯一的标准化资料编辑中心。
- 统一出生信息、现居地、时间可信度、历史事件等字段。
- 输出稳定的档案摘要。

### 第 3 步：报告页重做

- 报告页保持正式交付感。
- 固定封面、目录、基础档案、时间轴、八字、紫微、姓名、总结、注释、附录。
- 页面不能再像控制台。

### 第 4 步：结果页和 PDF 统一

- 页面预览与 PDF 章节结构保持同构。
- PDF / PNG 导出稳定。
- 人工批注区保留。

## 13. 验收标准

1. 用户进入首页后，只看到输入与预览，不会被复杂入口干扰。
2. 档案页成为唯一的标准化资料编辑中心。
3. 报告页呈现正式交付件气质。
4. 八字、紫微、姓名都遵循统一摘要结构。
5. 时间轴和预警区在报告中可见。
6. PDF 和网页预览结构一致。
7. 旧功能不回退，主流程稳定可用。

## 14. 结论

这套目标的核心不是继续扩张命理能力，而是先把产品形态收束成一条稳定主线：

`输入 -> 档案 -> 报告 -> 导出 -> 注释`

后续所有页面、章节和导出都应围绕这条主线展开。

## 15. 前端执行清单

这一节把目标进一步落到页面级别，作为后续修改首页、档案页、报告页时的直接依据。

### 15.1 首页 `HomeView`

#### 保留

- 基础信息输入
- 出生时间与精度设置
- 出生地与现居地
- 当前关注重点
- 历史事件摘要
- 个人数据卡预览
- 档案完整度与时间可信度

#### 新增

- 仅一个主按钮：`生成档案并进入报告`
- 明确的流程提示：`输入 -> 预览 -> 生成报告`
- 简短的状态块：档案完整度、时间可信度、最近保存状态

#### 砍掉

- 八字完整分析入口
- 紫微完整分析入口
- 姓名完整分析入口
- 多入口堆叠式布局
- 娱乐化文案和杂项跳转

#### 页面原则

- 只做入口收口和数据预览
- 不承担完整结果展示
- 不让分析入口抢主流程

### 15.2 档案页 `ProfileView`

#### 保留

- 姓名、性别、出生时间、出生地、时区、经纬度
- 公历 / 农历切换
- 闰月标记
- 时辰精度
- 未知时辰兜底
- 真太阳时开关
- 现居地
- 当前关注重点
- 历史关键事件

#### 新增

- 档案完整度提示
- 时间可信度提示
- 当前补全状态
- 统一的档案摘要区

#### 砍掉

- 正式报告正文
- 长篇命理解释
- 结果页式的排版
- 复杂入口堆叠

#### 页面原则

- 作为唯一的标准化资料编辑中心
- 所有字段都服务于后续报告生成
- 只输出稳定、可复用的基础档案契约

### 15.3 报告页 `ReportView`

#### 保留

- 封面
- 目录
- 个人基础档案
- 时间轴
- 八字总览
- 紫微总览
- 姓名分析
- 综合总结
- 人工注释区
- 附录与免责声明

#### 新增

- 主动预警区
- 章节锚点导航
- 结构摘要卡
- 与 PDF 同构的章节语法

#### 砍掉

- 表单编辑能力
- 控制面板式布局
- 复杂调参入口
- 和报告无关的工具内容

#### 页面原则

- 作为正式交付件
- 所有章节遵循 `先摘要、再结构、后解释`
- 网页预览与 PDF 保持同构

### 15.4 统一规范

#### 数据分层

- 基础信息层：姓名、性别、出生时间、出生地、历法、闰月、时区、经纬度、现居地、关注重点、历史事件
- 结果层：八字摘要、紫微摘要、姓名摘要
- 解释层：时间可信度、关键年份、风险提示、人工注释、综合结论

#### 内容语法

所有页面都使用同一套表达顺序：

1. 先摘要
2. 再结构
3. 后解释

#### 视觉语法

- 暖米白、浅砂金、深墨色、低饱和青绿色
- 档案感、纸张感、报告感、正式、克制、可信
- 禁止紫色霓虹、玄学闪光、娱乐插画、门户式密集排版

### 15.5 后续改动顺序

1. 先整理首页，让它只剩输入和预览。
2. 再整理档案页，让它变成唯一的标准化编辑中心。
3. 再整理报告页，让它成为正式交付件。
4. 最后让网页报告和 PDF 模板同构。

### 15.6 验收方式

- 首页进入后，主路径清晰，没有多入口干扰。
- 档案页可以稳定保存和回填基础资料。
- 报告页可以展示完整章节且不再像控制台。
- 时间轴、预警区、人工注释区都可见。
- PDF 与网页报告结构一致。
- 旧功能不回退，主线稳定。
