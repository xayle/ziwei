# 命理工具平台项目计划 v1.1

版本：v1.1
更新日期：2026-06-15
范围：基础档案、八字、紫微斗数、姓名学、跨模块通用能力

## 目标

把当前产品继续收敛为“个人档案型命理工具”，核心链路保持为：

`填写基础信息 -> 生成个人档案 -> 八字/紫微/姓名分析 -> PDF/图片交付 -> 人工注释讲解`

本版本只做三件事：

1. 先补齐影响正确性和交付体验的缺口。
2. 再补齐能提升复盘价值的结构化能力。
3. 暂不扩展低频、重社交、重重构功能。

## 产品原则

1. 先保证输入可信，再谈结果丰富。
2. 所有输出都要能落到档案、摘要、报告或时间线中。
3. 每条结论都尽量标明置信度或推断来源。
4. 优先做增量增强，不做大一统重写。
5. 与 PDF 交付无关的低频功能后置。

## P0 必须补

### 1. 基础档案可信度

- 公历 / 农历切换
- 闰月处理
- 夏令时自动转换
- 出生时间精度
- 未知时辰兜底
- 出生地标准化到城市 / 经纬度 / 时区
- 现居地字段

### 2. 结果可信度

- 高置信 / 中置信 / 推断项标注
- 边界时间提示
- 规则来源说明
- 报告中统一展示置信度图例

### 3. 交付闭环

- 统一摘要页
- PDF 导出
- 图片 / 卡片导出
- 分享链接
- 版本信息
- 校验摘要
- 人工注释区

### 4. 四大模块最小可用输出

- 八字：四柱、十神、五行、日主强弱、用神、格局、刑冲合害、空亡、调候
- 紫微：命宫、身宫、十二宫总览、三方四正、四化追踪、星曜亮度
- 姓名：五格、三才、八字补益匹配、谐音风险、三才冲突警告
- 通用：历史事件输入、命盘时间线、档案版本记录

## P1 应该补

### 基础档案

- 多胞胎修正
- 多档案管理
- 历史关键事件输入
- 命盘时间轴

### 八字

- 大运起运精确到天
- 换运提示
- 流年 / 流月高亮
- 多流派用神对照
- 趋避建议的量化依据

### 紫微

- 三方四正可视化
- 四化追踪表
- 宫位联动热力图
- 特殊格局识别
- 大限与流年重叠宫位权重

### 姓名

- 八字补益匹配度
- 负面谐音检测
- 字形 / 字义 / 字音评分
- 改名前后对比

### 通用

- 八字 + 紫微综合解读
- 预警栏
- 结果收藏

## P2 可选补

- 流日 / 流时
- 纳音分析
- 重名率查询
- 法律提示
- 社交卡片增强
- 多人对比
- 暗色模式
- 字体大小偏好

## 现在最该砍掉或后置的内容

- 公共社区化
- 强社交化产品方向
- 过度 AI 长文化
- 大一统重构
- 高复杂但低频功能扩张

## 推荐开发顺序

### Batch 1

- 基础档案可信度补齐
- 统一摘要页
- PDF / 分享 / 版本信息

### Batch 2

- 八字最小闭环增强
- 紫微最小闭环增强
- 姓名最小闭环增强

### Batch 3

- 历史事件与时间线
- 预警与复盘能力
- 报告结构继续优化

### Batch 4

- 高阶流派能力
- 低频扩展功能
- 体验型增强

## 验收标准

- 用户输入基础信息后，能生成完整个人档案
- 八字 / 紫微 / 姓名三条主线都有可读输出
- 报告支持摘要、版本、校验、注释、PDF 导出
- 结果能明确区分高置信、推断项和待验证项
- 现有主链路不被破坏
